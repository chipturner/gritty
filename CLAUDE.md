## What is gritty

gritty provides persistent TTY sessions over Unix domain sockets. Single binary, tmux-like CLI:
- `gritty server` — starts the server (self-daemonizes by default, returns after socket is bound). Alias: `s`. `-f`/`--foreground` runs in the foreground (for debugging or process managers).
- `gritty new-session <host[:name]>` — creates a persistent session and auto-attaches. `<host>` is required: a connection name from `gritty connect`, or `local` for the local server. Optional `:name` creates a named session. Auto-starts the server if none is running: `local` runs `gritty server`; other hosts run `gritty connect <host>`. Alias: `new`. `-A`/`--forward-agent` forwards local SSH agent. `-O`/`--forward-open` forwards URL opens to local machine. `--no-redraw` suppresses Ctrl-L redraw. `--no-escape` disables escape sequences. `--no-oauth-redirect` disables OAuth callback tunneling. `--oauth-timeout <seconds>` sets OAuth callback accept timeout (default: 180). `-w`/`--wait` waits indefinitely for the server instead of giving up after retries.
- `gritty attach <host:session>` — attaches to a session, detaches other clients. Waits indefinitely for the server if not yet running. Alias: `a`. `-A`/`--forward-agent` forwards local SSH agent. `-O`/`--forward-open` forwards URL opens to local machine. `--no-redraw` suppresses Ctrl-L redraw. `--no-escape` disables escape sequences. `--no-oauth-redirect` disables OAuth callback tunneling. `--oauth-timeout <seconds>` sets OAuth callback accept timeout (default: 180).
- `gritty tail <host:session>` — read-only stream of a session's PTY output (like `tail -f`). Alias: `t`. Does not detach the active client. Multiple tails can watch simultaneously. Streams raw PTY output -- not useful for TUI apps.
- `gritty send [--session host:session] [--stdin] <files...>` — sends files to a paired receiver. Inside a session (GRITTY_SOCK set), connects to svc socket directly. Outside a session, registers on all active sessions across all daemons and pairs with whichever gets a counterpart first. `--session` targets a specific session. `--stdin` reads data from stdin instead of files (sent as a single file named "stdin"; mutually exclusive with file arguments).
- `gritty receive [--session host:session] [--stdout] [dir]` — receives files from a paired sender. Destination dir defaults to cwd. Same routing and auto-detection as send. `--stdout` writes received data to stdout instead of creating files (no progress output).
- `gritty open <url>` — opens a URL on the local machine (for use inside gritty sessions with `--forward-open`). Reads `GRITTY_SOCK` from env.
- `gritty local-forward <port>` — forward a TCP port from session to client. Port spec: `8080` (same on both ends) or `8080:3000` (listen:target). Runs inside a session (`GRITTY_SOCK` required). Ctrl-C stops the forward. Alias: `lf`
- `gritty remote-forward <port>` — forward a TCP port from client to session. Same port spec syntax. Alias: `rf`
- `gritty connect <destination>` — SSH tunnel: self-backgrounds, auto-starts remote server, sets up tunnel, prints socket path and returns. Destination is `[user@]host[:port]`. Alias: `c`. `-n <name>` overrides connection name (defaults to hostname). `-o`/`--ssh-option` adds extra SSH options (repeatable). `--no-server-start` skips auto-starting the remote server. `-f`/`--foreground` keeps it in the foreground. `--dry-run` prints the SSH commands instead of running them.
- `gritty disconnect <name>` — tears down an SSH tunnel by connection name. Alias: `dc`
- `gritty tunnels` — lists active SSH tunnels with name/destination/status. Alias: `tun`
- `gritty list-sessions <host>` — lists active sessions with id/name/PTY/PID/status. Aliases: `ls`, `list`
- `gritty kill-session <host:session>` — kills a specific session
- `gritty protocol-version` — prints the protocol version number (integer)
- `gritty socket-path` — prints the default server socket path. Alias: `socket`
- `gritty kill-server <host>` — kills server and all sessions
- `gritty info` — shows diagnostics: version, config file path/status, socket paths, server status/session count, log file paths, active tunnels with status/pid/log paths
- `gritty config-edit` — opens config file in `$VISUAL`/`$EDITOR`/`vi` (creates from embedded template if missing)
- `gritty completions <shell>` — generates shell completions (bash, zsh, fish, elvish, powershell). E.g. `gritty completions bash > /etc/bash_completion.d/gritty`

Sessions get auto-incrementing integer IDs (0, 1, 2...) with optional human-friendly names via `host:name`.

Global options: `--ctl-socket <path>` overrides the default server socket path. `-v`/`--verbose` enables debug logging.

Client commands (`new-session`, `attach`, `tail`, `list-sessions`, `kill-session`, `kill-server`) accept an optional positional `<host[:session]>` argument. `<host>` is a connection name from `gritty connect`, or `local` for the local server. `<session>` (after the colon) identifies the session by name or ID (e.g., `gritty attach local:work`, `gritty new devbox:myproject`). The target is required unless `--ctl-socket` is provided (which overrides host resolution). Commands that require a session (`attach`, `tail`, `kill-session`) show available sessions if the session part is omitted. `send`/`receive` keep the target optional for in-session use. `gritty connect local` is rejected -- "local" is reserved for the local server keyword.

Similar to Eternal Terminal but socket-based. Sessions are persistent (shell survives client disconnect). A background server manages multiple sessions over a single socket.

## Architecture

Single-socket architecture: all communication (control AND session relay) goes through one server socket. Clients connect, perform a Hello/HelloAck version handshake, send a control frame declaring intent, server routes accordingly.

Seven modules behind a lib crate (`src/lib.rs` -- also hosts `collect_env_vars()`, `spawn_channel_relay()`, and `handshake()` helper) with a thin binary entry point (`src/main.rs`):

- **`security`** — Shared security utilities. `secure_create_dir_all` (0700 dirs, ownership validation, symlink rejection). `bind_unix_listener` (TOCTOU-safe stale socket handling, 0600 permissions). `verify_peer_uid` (SO_PEERCRED check). `checked_dup` (returns `OwnedFd`). `clamp_winsize`. All socket/directory creation MUST go through this module.

- **`config`** — TOML config file support (`$XDG_CONFIG_HOME/gritty/config.toml`). `ConfigFile` struct with `[defaults]` and `[host.<name>]` sections. `config_path()` returns the config file path. `DEFAULT_CONFIG` embeds the repo-root `config.toml` template via `include_str!`. `ConfigFile::load()` reads and parses, defaulting on missing/malformed. `resolve_session(host)` merges defaults + host overrides into `SessionSettings` (forward_agent, forward_open, no_escape, no_redraw, oauth_redirect, oauth_timeout). `resolve_connect(host)` merges into `ConnectSettings` (session settings + ssh_options + no_server_start). Precedence: CLI > host > defaults > built-in. SSH options use append semantics (host-specific first, then defaults; SSH first-match gives host priority). Unknown keys ignored for forward-compatibility.

- **`protocol`** — `Frame` enum with handshake types (Hello/HelloAck), session relay types (Data/Resize/Exit/Detached/Ping/Pong/Env/AgentForward/AgentOpen/AgentData/AgentClose/OpenForward/OpenUrl/TunnelListen/TunnelOpen/TunnelData/TunnelClose/SendOffer/SendDone/SendCancel/PortForwardListen/PortForwardReady/PortForwardOpen/PortForwardData/PortForwardClose/PortForwardStop), control request types (NewSession/Attach/Tail/ListSessions/KillSession/KillServer/SendFile), and control response types (SessionCreated/SessionInfo/Ok/Error). `PROTOCOL_VERSION: u16 = 1` constant for handshake negotiation. `SessionEntry` struct carries per-session metadata (id, name, pty_path, shell_pid, created_at, attached, last_heartbeat). `SvcRequest` enum (`OpenUrl=1`, `Send=2`, `Receive=3`, `PortForward=4`) -- 1-byte wire discriminator for the per-session svc socket; `from_byte(u8) -> Option<Self>` and `to_byte(self) -> u8`. Custom tokio-util `Encoder`/`Decoder`. Wire format: `[type: u8][length: u32 BE][payload]`. Handshake: `0x16` Hello (2-byte u16 BE version), `0x24` HelloAck (2-byte u16 BE version). Session relay: `0x01` Data, `0x02` Resize, `0x03` Exit, `0x04` Detached, `0x05` Ping, `0x06` Pong, `0x07` Env, `0x08` AgentForward, `0x09` AgentOpen, `0x0A` AgentData, `0x0B` AgentClose, `0x0C` OpenForward, `0x0D` OpenUrl, `0x0E` TunnelListen, `0x0F` TunnelOpen, `0x17` TunnelData, `0x18` TunnelClose, `0x19` SendOffer, `0x1A` SendDone, `0x1B` SendCancel, `0x1C` PortForwardListen, `0x1D` PortForwardReady, `0x1E` PortForwardOpen, `0x1F` PortForwardData, `0x26` PortForwardClose, `0x27` PortForwardStop. Control requests: `0x10` NewSession, `0x11` Attach, `0x12` ListSessions, `0x13` KillSession, `0x14` KillServer, `0x15` Tail, `0x25` SendFile. Control responses: `0x20` SessionCreated, `0x21` SessionInfo, `0x22` Ok, `0x23` Error.

- **`daemon`** — Listens on a single socket (`$XDG_RUNTIME_DIR/gritty/ctl.sock` or `/tmp/gritty-$UID/ctl.sock`). Accept loop requires Hello handshake first (5s timeout), responds with HelloAck (negotiated version = min of client and server), then reads control frame (5s timeout). Non-Hello first frame gets `Frame::Error`. All sends use `timed_send()` helper (5s timeout, logs warning on failure). Manages sessions in a `HashMap<u32, SessionState>` where `SessionState` holds `JoinHandle` + `Arc<OnceLock<SessionMetadata>>` + `mpsc::UnboundedSender<ClientConn>` for client handoff + optional name. Auto-incrementing `next_id` counter. Session names are validated to reject ASCII control characters (prevents tab/newline corruption of the SessionInfo wire format). Session resolution: name match first, then numeric id parse. Handles `NewSession` (validate name, allocate id, compute `agent-{id}.sock` + `svc-{id}.sock` paths relative to ctl_path's parent dir, create channel, spawn server with both paths, send SessionCreated, hand off framed connection as `ClientConn::Active`), `Attach` (resolve session, send Ok, hand off as `ClientConn::Active`), `Tail` (resolve session, send Ok, hand off as `ClientConn::Tail`), `SendFile` (resolve session, send Ok, extract raw `UnixStream` from `Framed` via `into_inner()`, hand off as `ClientConn::Send`), `ListSessions`, `KillSession`, `KillServer`. For Attach/Tail, connection handoff is skipped if Ok send fails. Reaps finished sessions before each operation. Signal handling: SIGTERM/SIGINT trigger clean shutdown (abort sessions, remove socket + PID file). PID file written to `socket_dir()/daemon.pid` after socket bind.

- **`server`** — `SessionMetadata` struct with pty_path, shell_pid, created_at, `AtomicBool` attached flag, `AtomicU64` last_heartbeat. `ClientConn` enum wraps `Active(Framed)` vs `Tail(Framed)` vs `Send(UnixStream)` connections. `ManagedChild` wraps `tokio::process::Child` with process-group cleanup (`killpg(SIGHUP)` on drop). `run()` takes `mpsc::UnboundedReceiver<ClientConn>` + `Arc<OnceLock<SessionMetadata>>` + `PathBuf` (agent socket path) + `PathBuf` (svc socket path). Two per-session sockets: `agent-{id}.sock` (SSH agent protocol, stays separate) and `svc-{id}.sock` (unified service socket for open URLs, file transfer). A `broadcast::channel<TailEvent>(256)` distributes PTY output to tail clients. `tail_relay()` handles per-tail-client event loop (Data/Exit broadcast + Ping/Pong keepalive). `spawn_tail()` drains ring buffer to a new tail client, then spawns `tail_relay`. Deferred shell spawn: allocates PTY early, waits for first client, reads optional `Env` frame (2s timeout), then spawns login shell (`-l`) with `CWD=$HOME`, allowlisted env vars (TERM/LANG/COLORTERM/BROWSER only), `SSH_AUTH_SOCK` pointing to the agent socket, and `GRITTY_SOCK` pointing to the svc socket. File transfer and open URL: unified per-session svc socket (`svc-{id}.sock`) bound at session start. `spawn_svc_acceptor()` accepts connections, reads 1-byte `SvcRequest` discriminator, dispatches: `OpenUrl` reads URL (up to 4KB), `Send` parses manifest, `Receive` parses dest dir. `TransferState` enum: Idle/WaitingForReceiver/WaitingForSender/Active. When both sides arrive, `spawn_transfer_relay()` sends go signal (0x01) to sender, writes per-file header+data interleaved to receiver, sends `SendOffer`/`SendDone`/`SendCancel` notification frames to active client. `ClientConn::Send(UnixStream)` handles local-side commands routed through daemon. Agent forwarding: spawns an acceptor task on the agent socket; when a process connects, assigns a channel_id and sends `AgentOpen` to the client (if forwarding enabled); data relayed via `AgentData`; `AgentClose` on either side tears down the channel. Open forwarding: on `OpenForward` frame, sets `open_forward_enabled = true` (svc socket is always bound, no lazy socket creation). When `gritty open <url>` connects to the svc socket with `SvcRequest::OpenUrl` discriminator and writes a URL, server checks `open_forward_enabled` and if so relays as `OpenUrl` frame to client; if URL contains `redirect_uri`/`redirect_url` pointing to `localhost`/`127.0.0.1` and the port is in use, sends `TunnelListen { port }` before `OpenUrl` for OAuth callback tunneling. Client spawns `xdg-open`/`open` with the URL (fire-and-forget). Tunnel relay: single-connection, no channel_id; `TunnelOpen` from client triggers TCP connect to local port, bidirectional relay via `TunnelData`, `TunnelClose` tears down. Port forwarding: `SvcRequest::PortForward` on svc socket reads 5-byte header `[direction:u8][listen_port:u16 BE][target_port:u16 BE]`. Direction 0 = local-forward (server binds TCP, each connection gets a channel_id, sends `PortForwardOpen` to client), direction 1 = remote-forward (sends `PortForwardListen` to client, waits for `PortForwardReady`, then client sends `PortForwardOpen` per TCP connection). `PortForwardData`/`PortForwardClose` are per-channel bidirectional. Svc socket EOF triggers `PortForwardStop` and cleanup. Multiple concurrent TCP connections per forward, each with independent relay. Uses generic `spawn_channel_relay` (same as agent forwarding). `PortForwardState` tracks listener handle, channels HashMap, and svc watcher handle per forward. On client disconnect/takeover, agent channels, open forwarding flag, and tunnel state are cleared and reset. Ring buffer tracks evicted bytes (`ring_buf_dropped`); on reconnect, if output was lost, prepends `[gritty: N bytes of output dropped]` marker before flushing buffered content. Receives clients via channel (no per-session socket). Inner relay select includes `client_rx.recv()` for client takeover — new client gets the session, old client receives `Detached` frame. Replies `Pong` to `Ping` and updates `last_heartbeat` timestamp.

- **`connect`** — Self-backgrounding SSH tunnel wrapper for remote access. `Destination` struct parses `[user@]host[:port]`. `remote_exec` runs commands on the remote host via SSH. `tunnel_command` builds hardened SSH tunnel commands (ServerAliveInterval, ServerAliveCountMax, StreamLocalBindUnlink, ExitOnForwardFailure, ConnectTimeout, -N -T). `spawn_tunnel` + `wait_for_socket` (200ms poll, 15s timeout). `tunnel_monitor` background task watches the SSH child and respawns on transient failure (exit 255 = retry; rate limit 5 exits in 10s). `ensure_remote_ready` gets remote socket path + optional protocol version and conditionally starts server; returns `(String, Option<u16>)`. `run()` warns on version mismatch between local and remote. `REMOTE_PATH_PREFIX` includes `$HOME/bin:$HOME/.local/bin:$HOME/.cargo/bin:/usr/local/bin:/opt/homebrew/bin:$PATH` for macOS Homebrew compatibility. `ConnectGuard` (Drop) kills SSH child, removes socket/pid/lock/dest files, releases flock. `run()` takes `ConnectOpts` + `ready_fd: Option<OwnedFd>` and orchestrates: parse destination, compute connection name, probe lockfile for existing tunnel, acquire flock, ensure remote ready, spawn tunnel, wait for socket, write PID/dest files, signal readiness, spawn monitor, wait for SIGTERM. `disconnect()` sends SIGTERM to tunnel process (reads PID file), polls lockfile for exit, escalates to SIGKILL. `list_tunnels()` enumerates `connect-*.lock` files, probes each (lock + socket connectivity), cleans stale ones. `parse_host()` extracts hostname from a `user@host` destination string. `get_tunnel_info()` reads dest/pid files for a named tunnel (used by `info` command). Local socket at `$XDG_RUNTIME_DIR/gritty/connect-{connection_name}.sock` (or `/tmp` fallback). Per-tunnel files: `.sock`, `.pid`, `.lock` (flock), `.dest`.

- **`client`** — `NonBlockGuard` saves/restores stdin's `O_NONBLOCK` flag on drop (prevents breaking parent shell). `RawModeGuard` saves/restores terminal mode. `EscapeProcessor` implements SSH-style `~` escape sequences (detach/suspend/help). `run()` takes session id/name, initial framed connection, redraw flag, `ctl_path` for reconnect, `env_vars: Vec<(String, String)>`, `no_escape: bool`, `forward_agent: bool`, and `forward_open: bool`. On first relay, sends `Env` frame (if non-empty), then `AgentForward` (if `forward_agent` and `SSH_AUTH_SOCK` set), then `OpenForward` (if `forward_open`), then `Resize`; on reconnect sends `AgentForward` + `OpenForward` + `Resize` (env_vars cleared). Agent forwarding: on `AgentOpen`, connects to local `SSH_AUTH_SOCK` and spawns bidirectional relay tasks; `AgentData` routed via HashMap; `AgentClose` tears down channels. Open forwarding: on `OpenUrl`, spawns `xdg-open`/`open` with the URL. Tunnel forwarding: on `TunnelListen { port }`, synchronously binds local TCP port (ready before `OpenUrl` is processed), accepts one connection with 180s timeout, relays via `TunnelData`/`TunnelClose`. Port forwarding: on `PortForwardListen`, binds local TCP port, spawns acceptor task, sends `PortForwardReady`; on `PortForwardOpen` (local-forward), connects to target port, spawns channel relay; `PortForwardData`/`PortForwardClose` routed via HashMap; `PortForwardStop` aborts listener and channels. All agent channels, port forwards, and tunnel state dropped on disconnect. Sends `Ping` every 5s; if no `Pong` within 15s, treats connection as dead. On disconnect/timeout, auto-reconnects via `ctl_path` (connect, handshake, Attach, resume relay). Prints `[reconnecting...]` / `[reconnected]` during the loop. Ctrl-C (0x03 in raw mode) exits during reconnect. Handles `Detached` frame with `[detached]` message (no reconnect on detach). File transfer notifications: `SendOffer` displays `[gritty: receiving N files (size)]`, `SendDone` displays `[gritty: transfer complete]`, `SendCancel` displays `[gritty: transfer cancelled: reason]`. `format_size()` (pub) formats byte counts as B/KB/MB/GB. `tail()` is a simplified read-only variant: no raw mode, no stdin, no escape processor, no agent/open forwarding; just streams Data frames to stdout with Ping/Pong keepalive and auto-reconnect (sends `Frame::Tail` instead of `Frame::Attach`).

## Patterns

- **Single-socket connection handoff**: Daemon reads Hello handshake (5s timeout), sends HelloAck, then reads control frame (5s timeout) and routes. For NewSession/Attach, daemon transfers the `Framed<UnixStream>` to the session task via `mpsc` channel. Daemon no longer touches that socket.
- **AsyncFd + try_io**: PTY master and stdin are raw fds wrapped in `AsyncFd`. Reads/writes use `guard.try_io()` with would-block continuation (`Err(_) => continue`).
- **Framed codec**: Socket I/O uses `tokio_util::codec::Framed<UnixStream, FrameCodec>` with `SinkExt`/`StreamExt` from futures-util.
- **PTY lifecycle**: `openpty` then defer shell spawn until first client connects. Read optional `Env` frame, then fork with `pre_exec(setsid + TIOCSCTTY)` using `-l` (login shell) and `CWD=$HOME`. Drop slave in parent then relay on master. EIO means shell exited.
- **Deferred shell spawn**: PTY is allocated early but shell spawn waits for first client so the server can read the `Env` frame and apply env vars. First client feeds directly into the relay loop (no re-wait in the outer loop).
- **Client environment forwarding**: Client sends `Env` frame with TERM/LANG/COLORTERM before the first `Resize` on new sessions. Server applies these as env vars when spawning the shell. On reconnect/attach, no `Env` frame is sent (shell already running).
- **Protocol version handshake**: Every connection starts with Hello/HelloAck. Client sends `Hello { version: PROTOCOL_VERSION }`, server computes `negotiated = min(client, server)` and replies `HelloAck { version: negotiated }`. `handshake()` helper in `lib.rs` handles the client side. Daemon rejects non-Hello first frame with `Frame::Error`. All client entry points (new_session, attach, tail, server_request) and reconnect loops call `handshake()` before sending control frames.
- **Persistent sessions**: PTY spawns once after the first client connects. Client disconnect breaks the inner relay loop only; outer loop drains PTY output into a userspace ring buffer (VecDeque<Bytes>, 1MB cap) so the shell never blocks. Oldest chunks are evicted when the cap is exceeded (`ring_buf_dropped` tracks evicted bytes). On reconnect, if output was lost, a `[gritty: N bytes of output dropped]` marker is sent before flushing buffered content. On reconnect, buffered output is flushed to the new client before entering the inner relay.
- **Client takeover**: Inner relay loop also selects on `client_rx.recv()`. New client causes `Detached` to be sent to old client, then relay switches to new connection.
- **Process group cleanup**: `ManagedChild` drop sends `SIGHUP` to shell's process group via `killpg`.
- **Terminal state guards**: `RawModeGuard` restores terminal attrs, `NonBlockGuard` restores stdin flags. Drop order ensures `NonBlockGuard` outlives `AsyncFd`.
- **Self-daemonizing**: `gritty server` forks before creating the tokio runtime. Parent waits on a pipe for the child to signal readiness (after socket bind), then prints PID and exits. Child calls `setsid()`, redirects stdin to `/dev/null`, and redirects stdout/stderr to a `.out` file (for panic/library output capture). `--foreground` skips the fork. The daemon writes a PID file to `socket_dir()/daemon.pid`.
- **Daemon logging**: Background daemons write structured tracing to log files (`daemon.log` / `connect-{name}.log`) with timestamps, levels, file names, and line numbers. Raw stdout/stderr is captured to `.out` files (`daemon.out` / `connect-{name}.out`) for panics and unexpected library output. Both use `O_APPEND` for truncation resilience. Foreground mode logs to stderr with ANSI colors. Default log level is `gritty=info` (overridden by `-v` for debug or `RUST_LOG` env var).
- **Daemon signal handling**: Accept loop uses `tokio::select!` over `listener.accept()`, `SIGTERM`, and `SIGINT`. On signal, calls `shutdown()` which aborts all sessions, removes socket and PID file.
- **Server auto-start**: `new-session` auto-starts on connection failure: `local` runs `gritty server`; other hosts run `gritty connect <host>` to start the tunnel. No auto-start with `--ctl-socket` (explicit path, user manages it). After auto-start, retries connection up to 10 times (500ms apart). `--wait` retries indefinitely instead. `attach` does not auto-start -- it waits indefinitely for an existing server. Other commands (`tail`, `ls`, `kill-*`) fail immediately if no server is running.
- **Auto-attach**: After `NewSession` succeeds, the same connection transitions to session relay mode (client calls `client::run` with the existing framed connection).
- **SIGWINCH on resize**: Server sends `killpg(SIGWINCH)` via `tcgetpgrp()` (foreground process group, not shell pgid) after every `TIOCSWINSZ`, ensuring foreground apps redraw on attach even when terminal size hasn't changed.
- **Ctrl-L redraw on attach**: Client sends `\x0c` after initial resize to force shell/app redraw. Controlled by `redraw: bool` param to `client::run()`, disabled with `--no-redraw` CLI flag on attach.
- **Ping/Pong heartbeat**: Client sends `Ping` every 5s, server replies `Pong` immediately and updates `last_heartbeat` in `SessionMetadata`. If client gets no `Pong` within 15s, connection is considered dead and client enters auto-reconnect loop. Zero-payload frames, 5 bytes on wire each.
- **Auto-reconnect**: On heartbeat timeout or disconnect, client loops: sleep 1s, connect to `ctl_path`, handshake, send `Attach`, read response. Terminal stays in raw mode throughout. Ctrl-C (0x03) during reconnect exits. On success, relay resumes with `redraw: true`. On `Frame::Error` (session gone), exits cleanly.
- **Session ID resolution**: Given a string, try name match first, then parse as u32 for id match.
- **SSH-style escape sequences**: `~` after newline (or at session start) enters escape mode. `~.` detaches (clean exit, no reconnect), `~^Z` suspends client (SIGTSTP), `~?` prints help, `~~` sends literal `~`. Unrecognized command flushes tilde + byte to server. `EscapeProcessor` state machine with 3 states (Normal/AfterNewline/AfterTilde). Disabled with `--no-escape` CLI flag.
- **Self-backgrounding connect**: `connect` command forks via `daemonize()` (same pattern as `gritty server`), sets up SSH tunnel, signals readiness to parent (which prints socket path and exits), then manages the tunnel in the background. Socket named by hostname (`connect-{host}.sock`), overridable via `-n <name>`. `--foreground` keeps it in the foreground. Uses `flock()` lockfiles for race-free liveness detection. Six files per tunnel in socket dir: `.sock`, `.pid`, `.lock` (flock), `.dest` (destination string), `.log` (structured tracing), `.out` (raw stderr capture). Tunnel monitor respawns SSH on transient failure (exit 255); non-transient (auth, config) bails. `ConnectGuard` Drop ensures cleanup (kill SSH, remove files, release lock) on all exit paths. `setsid()` during daemonize makes the connect process the session+group leader; orphaned SSH children cleaned via `killpg()` on stale detection.
- **Tunnel lifecycle commands**: `disconnect <name>` sends SIGTERM to tunnel process, polls lockfile for exit (2s), escalates to SIGKILL. `tunnels` enumerates lock files, probes each via flock + socket connectivity, reports healthy/reconnecting/stale (stale auto-cleaned).
- **Lockfile-based liveness**: Each background connect holds an exclusive `flock()` on `connect-{name}.lock`. Kernel releases automatically on process exit (including SIGKILL). Non-blocking probe (`LOCK_EX | LOCK_NB`) distinguishes live vs dead tunnels without races.
- **SSH agent forwarding**: Server creates a per-session agent socket (`agent-{id}.sock`) and sets `SSH_AUTH_SOCK` in the shell env. When a process connects, server assigns a channel_id and sends `AgentOpen` to the client (if `AgentForward` was received). Client connects to local `SSH_AUTH_SOCK` and relays data via `AgentData` frames. `AgentClose` from either side tears down the channel. On client disconnect/takeover, all agent channels are cleared and forwarding flag is reset. New client re-sends `AgentForward` if `--forward-agent` is set.
- **URL open forwarding**: Server sets `GRITTY_SOCK` in the shell env (pointing to `svc-{id}.sock`). On `OpenForward` frame from client, sets `open_forward_enabled = true` (the svc socket is always bound). When `gritty open <url>` connects with `SvcRequest::OpenUrl` discriminator and writes a URL, server relays it as `OpenUrl` frame to client if open forwarding is enabled (silently ignored otherwise). Client spawns `xdg-open`/`open` with the URL (fire-and-forget). Simpler than agent forwarding: no channel IDs, no bidirectional relay — each connection is a single URL. On client disconnect/takeover, open forwarding flag is reset. Client sets `BROWSER=gritty open` in the `Env` frame when `--forward-open` is set.
- **OAuth callback tunnel**: When a URL arriving on the svc socket (via `SvcRequest::OpenUrl`) contains `redirect_uri`/`redirect_url` pointing to `localhost`/`127.0.0.1`, server extracts the port, probe-binds to confirm something is listening, then sends `TunnelListen { port }` before `OpenUrl`. Client synchronously binds the port (guarantees ready before the browser opens), accepts one connection (configurable timeout via `oauth_timeout`, default 180s), and relays bidirectionally via `TunnelData`. `TunnelClose` from either side tears down. No channel_id -- single connection only. Part of `--forward-open`; can be disabled with `--no-oauth-redirect` or `oauth-redirect = false` in config (client-side: on `TunnelListen`, sends `TunnelClose` back immediately). Server-side: `extract_redirect_port()` parses URLs, `port_in_use()` probe-binds, `TunnelEvent` enum channels TCP data to main relay. Client-side: `ClientTunnelEvent` enum channels accept/data/close events. Tunnel state cleared on client disconnect/takeover.
- **Tail broadcast**: PTY output is distributed to tail clients via `tokio::sync::broadcast`. Each tail connection gets its own spawned `tail_relay` task subscribed to the broadcast. The main relay loop sends `tail_tx.send(TailEvent::Data(chunk))` alongside the active client write. Tail clients also receive ring buffer contents on connect (recent history). Tail does NOT detach the active client -- multiple tails coexist with one active client. Shell exit broadcasts `TailEvent::Exit` to all tails.
- **File transfer** (`gritty send` / `gritty receive`): Uses the unified per-session svc socket (`svc-{id}.sock`, env `GRITTY_SOCK`), always bound at session start. One transfer at a time per session with rendezvous: sender writes `SvcRequest::Send` discriminator + manifest (file_count, per-file name+size), receiver writes `SvcRequest::Receive` discriminator + dest_dir + newline. Server pairs them, sends go signal (0x01) to sender, then relays: per-file header (name_len, name, size) interleaved with file data from sender to receiver. Sentinel (name_len=0) marks end. `SendOffer`/`SendDone`/`SendCancel` notification frames sent to active TTY client. In-session: connect to `GRITTY_SOCK` directly. Outside a session: `connect_send_sockets()` discovers all sessions via `discover_all_sessions()`, connects to all via `send_file_handshake()`, returns all streams. Callers write role-specific init data (manifest or dest_dir) on all streams, then `select_first_ready()` waits for whichever session gets paired first. Remaining streams are dropped (cleaning up server-side state). `--session host:session` overrides auto-detect for a specific session. Filenames sanitized to basename only (path separators stripped, `..` rejected).
- **Port forwarding** (`gritty local-forward` / `gritty remote-forward`): Uses the svc socket (`GRITTY_SOCK`) with `SvcRequest::PortForward` discriminator. Wire: `[0x04][direction: u8][listen_port: u16 BE][target_port: u16 BE]`. Response: `0x01` = success, `0x02` + error message = failure. Svc connection stays open; drop triggers teardown. Local-forward (direction=0): server binds TCP, assigns channel_id per connection via `spawn_pf_tcp_acceptor`, sends `PortForwardOpen` to client per connection. Remote-forward (direction=1): server sends `PortForwardListen` to client, client binds TCP and sends `PortForwardReady`, then `PortForwardOpen` per connection; server connects to local target port. `PortForwardData`/`PortForwardClose` are per-channel bidirectional. `PortForwardStop` sent on svc socket EOF (Ctrl-C) -- cleans up listener + all channels. Multiple concurrent TCP connections supported per forward (multi-channel with unique channel_ids, same pattern as agent forwarding). Uses generic `spawn_channel_relay<R, W>` which works with both Unix and TCP streams.
- **Host-based socket routing**: Most client commands accept an optional positional `<host[:session]>` target. `parse_target()` splits on the first `:` to extract host and optional session. `"local"` maps to the default control socket (local server). Other hosts map to `connect-{host}.sock`. `resolve_ctl_path()` handles priority: `--ctl-socket` always wins (target can be omitted entirely) > `"local"` = default socket > other host = connect socket > neither = error. Commands requiring a session (attach, tail, kill-session) show a friendly error with available sessions if the session part is omitted. `gritty connect local` is rejected ("local" is reserved). Exception: `send`/`receive` use `--session` flag instead of positional target, with auto-detection across all daemons when omitted.
- **Security invariants**: Daemon sets `umask(0o077)` at startup. Sockets are 0600, directories 0700. All `accept()` sites verify `SO_PEERCRED` UID. Frame decoder rejects payloads > 1 MB. Resize values clamped to 1..=10000. `/tmp` fallback directories validated for ownership (not symlinks, owned by current uid).

## Current Status

All modules implemented and tested. Full CLI with tmux-like ergonomics, single-socket architecture, self-daemonizing server, SSH tunneling, agent/URL forwarding, OAuth callback tunneling, file transfer, TOML config, and read-only tail.

## Development Notes

### Critical invariants
- **`security` module is load-bearing** — all socket binding and directory creation goes through it. Never use `UnixListener::bind` or `create_dir_all` directly.
- **Reap before lookup** — `reap_sessions()` MUST be called before any operation that resolves a session (Attach, KillSession, ListSessions). Stale sessions in the HashMap cause silent failures.
- **Channel closed check** — Before sending `Frame::Ok` for Attach, check `client_tx.is_closed()`. If true, the session died between reap and lookup; send `Frame::Error` instead.
- **`Stdio::from(OwnedFd)`** — server uses safe `Stdio::from()` instead of `Stdio::from_raw_fd()`. Don't reintroduce `FromRawFd` in server.rs.
- **Fork before tokio** — `daemonize()` MUST fork before creating the tokio runtime (no threads allowed across fork). `main()` is sync (no `#[tokio::main]`).

### Changing protocol/signatures
- **`Frame` enum changes** — adding variants requires updating: encoder, decoder, protocol tests, and all `match frame` sites in server.rs, client.rs, daemon.rs, main.rs.
- **`SessionInfo` wire format** — 7 tab-separated fields per line: `id\tname\tpty_path\tshell_pid\tcreated_at\tattached\tlast_heartbeat`. Changing `SessionEntry` fields requires updating both encoder and decoder in protocol.rs.
- **`server::run()` signature** — called directly by e2e tests (via `UnixStream::pair()` + channel) and spawned by daemon. Changing its signature requires updating both.

### Testing
- **Test-first for bug fixes** — write a failing test first, implement the fix, confirm it passes. Regression tests go in daemon_test.rs (for daemon races) or e2e_test.rs (for session/relay bugs).
- **E2e tests use socketpair + channel** — no socket files needed, no cleanup issues. `UnixStream::pair()` creates both ends, server side sent via `mpsc` channel to `server::run()`.
- **Daemon tests use a real socket** — each test gets its own `tempfile::tempdir()`. Tests pass `None` for `ready_fd`. All helpers perform Hello handshake via `do_handshake()`.
- **Daemon tests are timing-sensitive** — use `tokio::time::sleep` to wait for daemon/session binding. If tests flake, increase sleep durations.
- **SSH/socat tests need external tools** — `just test-ssh` requires `sshd` + localhost SSH; `just test-socat` requires `socat`. Both skip gracefully if missing.

### Style
- **Error handling in main.rs** — `main()` returns `()`. Errors print as `error: <message>` via `eprintln!`, no backtraces. Never use `-> anyhow::Result` on `main()` in a CLI tool.
